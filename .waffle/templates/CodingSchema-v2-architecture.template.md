## プレースフォルダー定義票

| プレースフォルダー | 記入する内容 |
|---|---|
| `{{タイトル}}` | documentId をそのまま設定してください。（JSON上のフィールド: content.title.title） |
| `{{レイヤーと依存方向.様式}}` | 技術方式。例: ポートとアダプター（ヘキサゴナル）（JSON上のフィールド: content.layers.style） |
| `{{レイヤーと依存方向.項目1.レイヤー}}` | レイヤー名。例: domain, application, ports, inbound adapter, outbound adapter（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.layers.items[].layer） |
| `{{レイヤーと依存方向.項目1.責務}}` | このレイヤーの責務を1文で。例: 外部I/O・外部ライブラリをport実装に閉じ込める（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.layers.items[].responsibility） |
| `{{レイヤーと依存方向.項目1.依存してよい先1}}` | 依存してよいレイヤー名を列挙（内向きのみ）。最内層は空。（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.layers.items[].mayDependOn[]） |
| `{{ディレクトリ構成.ツリー}}` | 正典ディレクトリツリー（テキスト）。`{package}` 等プレースホルダを使ってよい（プロダクト非依存）。（JSON上のフィールド: content.layout.tree） |
| `{{ディレクトリ構成.合成ルート}}` | 結線（DI）を置く場所を1文で。例: inbound adapter の起動点にのみ置く（JSON上のフィールド: content.layout.compositionRoot） |
| `{{概念実現形.項目1.概念}}` | 固定語彙から選ぶ（spec の specKind と対応）。（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.conceptPlacement.items[].concept） |
| `{{概念実現形.項目1.配置}}` | layout のツリー上の配置（相対パス）。例: application/usecases（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.conceptPlacement.items[].placement） |
| `{{概念実現形.項目1.形}}` | この概念の実現方針を決定レベルで（正確な署名は書かない）。例: エントリメソッド1つ・ドメインは port 経由で呼ぶ（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.conceptPlacement.items[].pattern） |
| `{{規約.項目1.種別}}` | 種別。必須 / 禁止 / 推奨のいずれか。（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.rules.items[].level） |
| `{{規約.項目1.規約}}` | 依存方向・DIの受け取り方・エラー伝播（結果型/例外の使い分け）等、横断的な決定ルールの内容。例: 依存は内向きのみ（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.rules.items[].rule） |
| `{{サブドメイン別の厚み.項目1.Category}}` | subdomain spec の Category。（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.thicknessBySubdomain.items[].category） |
| `{{サブドメイン別の厚み.項目1.実装の厚み}}` | 実装の厚み方針を1文で。例: ドメインモデルで厚く実装する（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.thicknessBySubdomain.items[].thickness） |
