| `{{判断基準.遷移1.遷移先}}` | 遷移先ノードのid。stagesで宣言済みのidの中から選ぶ。（JSON上のフィールド: content.decisionCriteria.transitions[].to） |

| `{{実例}}` | 架空の題材を使い、この概念を具体的に当てはめた実例を記述してください。特定の実在企業名は避けてください。（JSON上のフィールド: content.examples.text） |
| `{{アンチパターン.項目1.アンチパターン}}` | アンチパターン名。何が誤りかが分かる短い名詞句で。例: 貧血ドメインモデル（配列。この形式の行を必要な数だけ繰り返す）（JSON上のフィールド: content.antiPatterns.items[].name） |
| `{{アンチパターン.項目1.問題点}}` | その誤りが引き起こす具体的な問題を1文で。（JSON上のフィールド: content.antiPatterns.items[].problem） |
| `{{アンチパターン.理由}}` | itemsが空の場合、この概念によくある誤りが無い理由を書いてください。itemsに中身がある場合は空文字でよい。（JSON上のフィールド: content.antiPatterns.emptyReason） |
| `{{出典・根拠の透明性.出典}}` | 内容の出典（著者名・書籍/論文名・年等）と、要約/再構成であり直接引用でないことを明記してください。（JSON上のフィールド: content.provenance.source） |
| `{{出典・根拠の透明性.留保事項}}` | 調査・検証の過程でまだ確証が得られていない留保事項があれば書いてください（例: 特定の定式化が検証で却下された等）。無ければ空文字。（JSON上のフィールド: content.provenance.caveats） |
