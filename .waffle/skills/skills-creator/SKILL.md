## 目的

ベストプラクティスに従ったフォルダ構成とSKILL.mdを備えた新しいSkillsを、プレースフォルダー付きスケルトンから量産できるようにすること。

## 役割

- スキル設計のコンサルタントとして要件をヒアリングする
- ベストプラクティスのフォルダ構成を遵守した実装者として動く
- SKILL.mdの各セクション（目的・役割・実行手順・ガードレール）を適切に埋めるファシリテーターとして振る舞う
- 作成後はスキルの動作確認方法をユーザーに伝える

## 実行手順

### Step 1: 要件のヒアリング

以下を確認する。不明な項目はユーザーに質問する。

1. **スキル名** — 英小文字・ハイフン区切り（例: `code-reviewer`）
2. **トリガー条件** — どんな言葉・状況でこのスキルが発動すべきか
3. **スコープ** — プロジェクト専用（`.claude/skills/`）かグローバル（`~/.claude/skills/`）か
4. **テンプレートの要否** — 出力ドキュメントを生成するスキルか、会話だけで完結するスキルか
5. **追加フォルダの要否** — `examples/`・`scripts/`・`agents/` が必要か

### Step 2: プレースフォルダー定義票を参照する

`.claude/skills/skills-creator/references/skill-template.md` を読み込み、各プレースフォルダーの意味を把握する。
`.claude/skills/skills-creator/references/folder-structure.md` を読み込み、使用するフォルダ構成を決定する。

#### プレースフォルダー定義票

| プレースフォルダー | 説明 |
|---|---|
| `{{スキル名}}` | ヒアリングで確認したスキル名。英小文字・ハイフン区切り。 |
| `{{トリガー条件の説明}}` | ユーザーがどんな言葉を使ったときにこのスキルが発動すべきかを具体的に記述する。「〜するとき」「〜と言われたとき」の形。 |
| `{{このスキルが何を達成するか}}` | ユーザー視点のゴールを1〜2文で表す。 |
| `{{役割N}}` | Claudeがこのスキル実行中にとる立場。ファシリテーター・実装者・アドバイザーなど文脈に合ったものを選ぶ。 |
| `{{最初のステップ名}}` | Step1の内容を端的に表すタイトル。「モード判定」「要件確認」「入力解析」など。 |
| `{{プレースフォルダー名}}` | そのスキルが使うテンプレートのプレースフォルダー名。`{{}}` で囲む。 |
| `{{何を書くか}}` | そのプレースフォルダーに何を書けばよいかをClaudeへの指示として記述する。 |
| `{{テンプレートファイル名}}` | `references/` に置くテンプレートファイルの名前（拡張子なし）。 |
| `{{ドキュメントの保存パス}}` | 生成するドキュメントをどこに保存するか。例: `docs/{{テーマ}}.md` |
| `{{してはいけないこと}}` | ガードレールの内容。品質・安全・ユーザー体験を守るための制約。 |

### Step 3: フォルダ構成を決定して作成する

`folder-structure.md` を参照し、ヒアリング内容に応じてミニマム構成かフル構成かを選ぶ。

以下のフォルダ・ファイルを作成する（テンプレートが不要な場合は `references/` を省略）。

```
{{スコープ}}/skills/{{スキル名}}/
├── SKILL.md
└── references/           # テンプレートが必要な場合のみ
    └── template.md
```

### Step 4: SKILL.md を生成する

`references/skill-template.md` を読み込み、プレースフォルダー定義票に従って埋める。
生成した内容を `{{スコープ}}/skills/{{スキル名}}/SKILL.md` として保存する。

### Step 5: テンプレートファイルを生成する（必要な場合のみ）

ユーザーが「出力ドキュメントを生成するスキル」を望む場合、スケルトンテンプレートを作成する。
テンプレートにはプレースフォルダーを配置し、各プレースフォルダーの説明をSKILL.mdのプレースフォルダー定義票に追記する。
保存先: `{{スコープ}}/skills/{{スキル名}}/references/template.md`

### Step 6: 完成確認

作成したファイル一覧を表示し、スキルの呼び出し方をユーザーに伝える。

## ガードレール

- `SKILL.md` は必ずスキルフォルダのルートに置く。サブフォルダには置かない
- テンプレートファイルは `assets/` ではなく `references/` に置く
- 不要なフォルダは作らない。使うものだけ作る
- プレースフォルダーを埋める前に必ず定義票を参照する。定義票なしで埋めない
- スキル名は英小文字・ハイフン区切りに統一する（スペース・アンダースコア不可）
- ユーザーが要件を明確にしていない場合は作成を開始しない。必ずヒアリングを先に完了させる

## テンプレートファイル

- `.claude/skills/skills-creator/references/skill-template.md` — SKILL.md スケルトン
- `.claude/skills/skills-creator/references/folder-structure.md` — フォルダ構成ベストプラクティス
